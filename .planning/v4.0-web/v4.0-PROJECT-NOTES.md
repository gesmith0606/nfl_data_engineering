# v4.0 Web Platform — Project Notes

## Tech Stack Decisions

| Layer | Technology | Rationale |
|-------|-----------|-----------|
| Frontend | Next.js 14+ (App Router) | SSR for SEO, React Server Components, API routes |
| Styling | Tailwind CSS v4 | Utility-first, dark mode support, responsive design |
| Backend API | FastAPI (Python 3.11+) | Same language as existing pipeline, async, auto OpenAPI docs |
| Database | PostgreSQL via Supabase (free tier) | Managed, REST/realtime built-in, upgrade path to RDS |
| Frontend hosting | Vercel | Zero-config Next.js deployment, preview deploys per PR |
| Backend hosting | AWS Lambda (API Gateway) | Low cost at MVP traffic, same AWS account as S3 |
| Auth (future) | Supabase Auth or NextAuth.js | Defer until Phase W7+ |

### Data Flow

1. **Pipeline (unchanged)**: Python scripts run weekly (GHA cron or manual)
   - Bronze ingestion → Silver transformation → Gold projections/predictions
   - Writes Parquet to `data/gold/projections/` and `data/gold/predictions/`

2. **DB Sync (Phase W5)**: After pipeline writes Parquet, a sync script loads results into PostgreSQL
   - `scripts/sync_gold_to_db.py` reads latest Parquet, upserts into `projections` and `predictions` tables
   - Until Phase W5, FastAPI reads directly from Parquet files

3. **API Serving**: FastAPI reads from PostgreSQL (or Parquet fallback) and returns JSON
   - Stateless — no session, no user state in MVP

4. **Frontend Rendering**: Next.js fetches from FastAPI at build time (SSG) or request time (SSR)
   - Projections pages: SSR with 5-minute ISR revalidation
   - Player detail pages: SSR on demand

## Design Considerations

### Mobile-First
- Primary audience checks projections on phones during draft or game day
- Table must scroll horizontally on small screens; key columns (name, position, points) always visible
- Touch targets min 44px; no hover-only interactions

### Dark Mode
- Default to system preference via `prefers-color-scheme`
- Manual toggle persisted in localStorage
- Dark mode is primary — most fantasy apps use dark themes

### NFL Team Colors
- Accent colors per team on player detail and matchup cards
- Source: static map of 32 teams → primary/secondary hex codes
- Used for: team badges, matchup card borders, position group headers

### SEO
- Server-rendered projection pages with structured data (JSON-LD)
- URL structure: `/projections/week-1-2026`, `/predictions/week-1-2026`, `/players/josh-allen`
- Meta descriptions with player names and point projections for Google snippets
- Open Graph tags for social sharing (projection card images deferred)

| Backtest results | `scripts/backtest_projections.py` | MAE, RMSE, correlation, bias per position |
| Kicker projections | `src/kicker_projection.py` → `generate_kicker_projections()` | kicker stats + projected_points |
