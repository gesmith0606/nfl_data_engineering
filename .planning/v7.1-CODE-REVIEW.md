## CRITICAL

### C-01 — `get_comparison` uses CWD-relative path instead of `DATA_DIR`

**File:** `web/api/services/projection_service.py:396`  
**Phase:** 73

```python
silver_root = Path("data/silver/external_projections")
```

Every other data-path in the web API layer is anchored via `web/api/config.py`:

```python
DATA_DIR = Path(os.getenv("NFL_DATA_DIR", str(_PROJECT_ROOT / "data")))
```

`_PROJECT_ROOT` is resolved from `Path(__file__)`, so it is always correct regardless of the working directory uvicorn is started from. The bare `Path("data/...")` form resolves relative to CWD. On Railway the process may not be started from repo root, and the `NFL_DATA_DIR` env var override (which operators can set to a mounted volume path) is silently ignored here.

`news_service.py`, `projection_service.py` (all other functions), and the Parquet helper `GOLD_PROJECTIONS_DIR` all import from `web/api/config.py`. The `get_comparison` function is the sole outlier.

**Fix:** Replace the two bare `Path("data/...")` literals at lines 396-397 with config-anchored paths:

```python
from ..config import DATA_DIR
silver_root = DATA_DIR / "silver" / "external_projections"
```

### H-03 — `sleeper_user_login` raises HTTP 404 on Sleeper outage (D-06 violation)

**File:** `web/api/routers/sleeper_user.py:99-102`  
**Phase:** 74

```python
user_payload = fetch_sleeper_json(user_url)   # returns {} on outage
user = _to_user(user_payload)                 # returns None for empty dict
if user is None:
    raise HTTPException(status_code=404, ...)  # always fires on outage
```

`fetch_sleeper_json` is explicitly fail-open: it returns `{}` on any network error. `_to_user({})` returns `None` because `{}.get("user_id")` is falsy. The endpoint then unconditionally raises 404.

The docstring says "D-06 fail-open: Sleeper outage returns 200 with empty leagues list." That is true only for the leagues sub-call (line 108). The user lookup itself is not fail-open: a Sleeper outage produces a 404, which the frontend would interpret as "user not found" and show a red error banner — it cannot distinguish "Sleeper is down" from "wrong username."

**Fix:** Return 503 (service unavailable) or a specific error code when `fetch_sleeper_json` returns `{}` for a user lookup, so the frontend can differentiate the two cases. Alternatively, return 200 with a sentinel `user=null` shape and document it as the fail-open state, then update the docstring.

### M-02 — `TeamWeeklyAggregator._aggregate_by_team` merges non-player counts with `df.at[]` inside `iterrows`

**File:** `src/sentiment/aggregation/team_weekly.py:480-489`  
**Phase:** 72

```python
for idx, row in df.iterrows():
    team = row["team"]
    if team in non_player_counts:
        df.at[idx, "coach_news_count"] = non_player_counts[team]["coach_news_count"]
        ...
```

This iterates every row of the team DataFrame (32 teams max) and mutates it with `df.at[]`. The performance impact is negligible (32 rows), but the pattern is fragile: `df.at[]` with chained indexing can trigger `SettingWithCopyWarning` if `df` is a slice. A vectorised merge is both safer and idiomatic:

```python
counts_df = pd.DataFrame.from_dict(non_player_counts, orient="index").reset_index()
counts_df = counts_df.rename(columns={"index": "team"})
df = df.merge(counts_df, on="team", how="left", suffixes=("", "_new"))
# fill NaN with 0 for the three count columns
```

This also removes the risk of triggering a SettingWithCopyWarning if the DataFrame is a view.

### M-04 — `_delta` function defined inside a `for` loop in `get_comparison`

**File:** `web/api/services/projection_service.py:482-487`  
**Phase:** 73

```python

```

`_delta` is defined inside the `for _ in wide.iterrows()` loop — except the loop itself follows `wide["delta_vs_ours"] = wide.apply(_delta, axis=1)` at line 489. Looking at the actual code:

def _delta(row):
    ours = row.get("ours")
    externals = [row.get(s) for s in ("espn", "sleeper", "yahoo") if pd.notna(row.get(s))]
    if pd.isna(ours) or not externals:

`_delta` is defined before the `iterrows` loop at line 498, so it is redefined on each call to `get_comparison`. Move it to module scope. Since `grouped` already has one row per `(player, source)`, the `aggfunc="first"` is redundant. Use `aggfunc="mean"` or `values="projected_points"` with the pre-aggregated input for clarity.

### L-02 — `_roster_provider_factory` uses `datetime.now().year` without `timezone.utc`

**File:** `src/sentiment/processing/pipeline.py` (inside `_build_extractor`)  
**Phase:** 71

```python
roster_provider = self._roster_provider_factory(
    season=datetime.now().year
)
```

`datetime.now()` returns a naive local datetime. For a process running on Railway (UTC) or a developer on US/Eastern, `.year` resolves identically on most days, but within the first hour of UTC Jan 1 it can return the prior year for US timezones. Use `datetime.now(timezone.utc).year` for consistency with the rest of the codebase, which universally uses UTC.

**Files:**

function relativeTime(isoString: string | null): string {
  if (!isoString) return '';
