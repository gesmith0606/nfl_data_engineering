# v8.0 Live Draft Co-Pilot тАФ Requirements

**Milestone goal:** A Claude Code-native live draft assistant for Sleeper, Yahoo, and ESPN drafts, run by the operator inside VS Code. Sleeper ships first; Yahoo and ESPN plug into the same adapter-based engine. Separate from the website chatbot.

## SLPR тАФ Sleeper Draft Data Layer (Phase 85)

- **SLPR-01** тАФ `src/sleeper_http.py` gains draft-endpoint helpers (`get_drafts_for_league`, `get_draft`, `get_draft_picks`, `get_traded_picks`), each preserving the existing fail-open contract (return `{}`/`[]` on any error).
- **SLPR-02** тАФ Resolve a live draft from a Sleeper username (+ season) alone: user тЖТ leagues тЖТ drafts тЖТ select active/most-recent; return `draft_id` + league scoring/roster settings + draft order.
- **SLPR-03** тАФ Build and cache a Sleeper `player_id` тЖТ projection-player map from `/v1/players/nfl`; map тЙе95% of skill-position picks; surface (never silently drop) unmatched picks.
- **SLPR-04** тАФ Typed `DraftState` + `PickEvent` models with unit tests against a recorded real-draft fixture.

## ENG тАФ Live Draft Engine (Phase 86)

- **ENG-01** тАФ Idempotent poll/diff loop: detect draft status, emit only new `PickEvent`s since last-seen `pick_no` per tick.
- **ENG-02** тАФ Live `DraftBoard` + every-team roster reconstruction; snake/linear order + on-the-clock slot; auto-detect user's slot and next pick number.
- **ENG-03** тАФ On-the-clock recommendations via `DraftAdvisor` (VORP, value tier vs ADP, roster need, bye conflict) + best-available-by-position from current projections.
- **ENG-04** тАФ Key-moment detection: value drop, positional run, reach/steal vs ADP, post-pick grade.
- **ENG-05** тАФ `DraftAdapter` protocol + `SleeperAdapter` reference impl; engine consumes only the adapter interface (no platform specifics leak into engine core). (D-08)

## SKILL тАФ Claude Code Draft Skill (Phase 87)

- **SKILL-01** тАФ `/draft-live` skill: start from `--platform` + username/draft_id, dispatch via `DraftAdapter`, auto-apply league scoring + roster format, begin polling тАФ no code edits required.
- **SKILL-02** тАФ Conversational + proactive loop (D-05): on-turn + key-moment alerts, plus ad-hoc Q&A (who to pick / best-at-position / compare / remaining needs).
- **SKILL-03** тАФ Operational polish: start/stop, resume mid-draft, paused/slow-draft tolerance, read-only auction awareness, docs in CLAUDE.md + cross-link from `draft-prep`.
- **SKILL-04** тАФ Manual-entry fallback (D-09): operator-typed picks drive the engine when no adapter is live, so any platform/draft is assistable.

## YH тАФ Yahoo Draft Adapter (Phase 88)

- **YH-01** тАФ Yahoo OAuth2: registered app, one-time consent, token store, automatic refresh-token rotation (~60-min expiry); secrets only from env/`.env`; clean re-auth on refresh failure.
- **YH-02** тАФ `YahooAdapter` conforming to `DraftAdapter`: resolve league/draft тЖТ poll `draft_results` тЖТ normalize тЖТ map players; conservative polling + backoff. Engine + skill run unmodified.
- **YH-03** тАФ Offline fixture tests (recorded `draft_results`, OAuth/HTTP mocked, no CI network), тЙе95% skill-position mapping; operator smoke against a real Yahoo draft.

## ESPN тАФ ESPN Draft Adapter, spike-gated (Phase 89)

- **ESPN-01** тАФ **[SPIKE]** Time-boxed feasibility spike on live in-progress ESPN pick capture (REST `mDraftDetail` latency vs Playwright/DOM watcher vs `espn_s2`/`SWID` cookie auth) тЖТ documented GO/NO-GO + chosen mechanism. Gates ESPN-02/03.
- **ESPN-02** тАФ **[gated, GO]** `EspnAdapter` conforming to `DraftAdapter` via the spike-chosen mechanism, behind a feature flag; тЙе90% skill-position mapping on a fixture.
- **ESPN-03** тАФ **[gated]** Tests (offline fixtures/recorded DOM) + operator smoke if GO; if NO-GO, record decision + verify manual-entry fallback (D-09) as the ESPN path. ToS/fragility caveats documented.
