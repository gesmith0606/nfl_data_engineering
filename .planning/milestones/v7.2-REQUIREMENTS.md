# v7.2 Data + Site Polish — Requirements

**Milestone goal:** Tighten what's already shipped — close half-done flows, kill tech debt, and lock in production reliability so v8.0 (PFF) can build on a clean foundation.

**Mode:** "Tighten each theme" — every requirement cut to its smallest defensible scope. Some categories are single-PR phases; others span 2-3 days.

---

## Active Requirements (v7.2)

### AWS Refresh + S3 Sync (AWS-XX)

Goal: AWS credentials + S3 layer back online so cloud workflows resume and PFF prep can begin.

- [ ] **AWS-01**: Operator can refresh AWS credentials and verify reads against `nfl-raw`, `nfl-refined`, `nfl-trusted` buckets from a fresh shell using `download_latest_parquet`.
- [ ] **AWS-02**: Local `data/bronze/`, `data/silver/`, `data/gold/` are one-shot synced to S3 (idempotent re-runs OK; deltas-only after first sync).
- [ ] **AWS-03**: At least one production read path (e.g. one Silver transformation script) is smoke-tested against S3 in CI; passes when AWS_* secrets are set.

### Sentiment Ingestion Depth (INGEST-XX)

Goal: Lift Phase 72 EVT-04 team-coverage floor (currently 9-12) by adding 2 high-yield sources. Lets CONTEXT D-04 tighten back toward 15-20 over time.

- [ ] **INGEST-01**: ESPN team RSS ingester lands under `data/bronze/sentiment/espn_team/season=YYYY/` partitioned by season; one Bronze JSON file per ingestion run; integrated into `daily-sentiment.yml` D-06 fail-open contract.
- [ ] **INGEST-02**: Yahoo Sports RSS ingester lands under `data/bronze/sentiment/yahoo_sports/season=YYYY/`; integrated into the same daily cron flow.
- [ ] **INGEST-03**: A fresh W17 + W18 backfill against the expanded source set produces ≥ 12 of 32 teams with non-zero events on `/api/news/team-events` (CONTEXT D-04 amended gate is 8; this verifies the floor lift, doesn't change the gate).

### Heuristic Consolidation (HEUR-XX)

Goal: Eliminate the 3-way duplicate heuristic that caused the v4.1-p3 QB-bias bug. Long-running tech debt; carried from v6.0 backlog.

- [ ] **HEUR-01**: Single canonical `compute_heuristic_projection()` function (or equivalent named convention) with golden-output test fixtures pinned per position (QB/RB/WR/TE) for at least 5 representative player-weeks each.
- [ ] **HEUR-02**: All 3 legacy callers (`generate_weekly_projections` in `projection_engine.py`, `generate_heuristic_predictions` in `player_model_training.py`, `compute_production_heuristic` in `unified_evaluation.py`) delegate to the canonical function. Behavioral equivalence proved by golden tests.
- [ ] **HEUR-03**: QB-bias regression test asserts QB heuristic does NOT NaN-propagate when `usage_multiplier` upstream is missing or stale (the v4.1-p3 root cause that consolidation closes).

### Data Freshness + Correctness (DQ-XX)

Goal: Make audit + deploy gate evidence forensically traceable. Catalog accumulated sanity-warning debt.

- [ ] **DQ-01**: Audit-script JSON outputs (`event_coverage.json`, `advisor_tools_72.json`, plus future audit JSONs) capture `script_sha` of the audit script at execution time so future re-runs can be distinguished from historical runs.
- [ ] **DQ-02**: Railway exposes `/api/version` returning `{git_sha, deployed_at, llm_enrichment_ready}` so the live gate can assert the running image matches the just-pushed commit (closes the 3-week silent-freeze failure mode that v7.1 hit).
- [ ] **DQ-03**: 27 pre-existing sanity warnings cataloged + triaged. Each warning gets one of 4 verdicts: KEEP (legitimate signal), UPGRADE (promote to CRITICAL), DELETE (stale check), RAISE-THRESHOLD (relax to current-state floor with documented why).

### Feature Audit + Gap Closure (UX-XX)

Goal: Walk every dashboard page; fix EmptyState + mobile regressions surfaced. NOT a redesign — pure gap closure.

- [ ] **UX-01**: Every dashboard route (PROJECTIONS, PREDICTIONS, LINEUPS, MATCHUPS, NEWS, LEAGUES, DRAFT, plus PLAYER detail) renders cleanly with EmptyState fallbacks where data is sparse. Audit document with per-page screenshots + verdicts committed under `.planning/milestones/v7.2-phases/[phase]/UX-01-AUDIT.md`.
- [ ] **UX-02**: Every dashboard route renders correctly at 375px mobile width (no horizontal scroll, tap targets ≥ 44px, font legibility ≥ 14px). Audit doc captures any regressions; fixes commit-tracked. Re-uses the design tokens + motion primitives shipped in Phase 62.

### Sleeper OAuth + Multi-User (AUTH-XX)

Goal: Replace cookie-only auth with refresh-token flow + persistent sessions. Minimum viable depth — not full enterprise auth.

- [ ] **AUTH-01**: User can sign in via Sleeper OAuth, receive a refresh token, and have their session persisted in a backend session table (SQLite if simpler, Postgres if already wired). Session cookie carries an opaque session ID, not a Sleeper user ID.
- [ ] **AUTH-02**: User's authenticated session survives a 24h gap (refresh token exchange works without re-auth). Refresh failure falls back gracefully to a re-auth prompt, not a 500.
- [ ] **AUTH-03**: Existing cookie-based `sleeper_user_id` flow (Phase 74 SLEEP-01) continues to work as a backwards-compat path. Phase-out + deletion deferred to v7.3.

### Advisor Tool Expansion (TOOL-XX)

Goal: Surface the new v7.1 features (external projections, Sleeper rosters, matchup data) to the chat advisor as native tools.

- [ ] **TOOL-01**: `getProjectionComparison({player_name})` tool returns the 4-source comparison table for a player (wraps `/api/projections/comparison`). Tool description guides the LLM to call this when users ask "what does ESPN say about X" or similar comparison questions.
- [ ] **TOOL-02**: `getMatchupAdvantages({team, week})` tool returns positional-rank advantages for a team-week (wraps `/api/teams/{team}/defense-metrics`). Tool description guides "matchup quality" / "how good is X's defense" queries.
- [ ] **TOOL-03**: `getUserLeagueRoster` tool is auth-aware — uses authenticated session (AUTH-01) when present, falls back to cookie-based `sleeper_user_id` (Phase 74) when not. No tool-list bifurcation; one tool, two paths.

### Deploy Hardening (DEPLOY-XX)

Goal: Close the gaps surfaced by today's 3-week silent freeze. Make future deploys self-validating.

- [ ] **DEPLOY-01**: Auto-rollback skips push when the failing commit touches `.github/workflows/**`, with a `::error` summary listing manual-rollback steps. Alternatively (operator preference): provision `ROLLBACK_PAT` repo secret with `workflow` scope and substitute it for `GITHUB_TOKEN` in the rollback push step.
- [ ] **DEPLOY-02**: Vercel chunk-fingerprint probe in `deploy-frontend` is promoted from warn-only to fail when fingerprint unchanged AND Railway `/api/version` (DQ-02) indicates Railway redeployed (asymmetric drift = real Vercel webhook freeze).
- [ ] **DEPLOY-03**: Structural test asserts the rebase+retry loop in `daily-sentiment.yml` push step (introduced in commit `aa4ac70`) survives future refactors. Test is part of the existing deploy-workflow suite.
- [ ] **DEPLOY-04**: Live gate's audit-JSON consumer asserts `script_sha` (DQ-01) field exists and matches a known-good audit-script commit (or main HEAD); rejects audit evidence produced by an unknown script revision.
