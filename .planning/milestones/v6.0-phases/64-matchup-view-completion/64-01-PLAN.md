  <name>Task 2: Write API-CONTRACT.md</name>
  <files>.planning/phases/64-matchup-view-completion/API-CONTRACT.md</files>
  <action>
Specify the three new endpoints that plans 64-02 and 64-03 will implement. For each, document: HTTP method + path, query params (with types + defaults + validation), response Pydantic schema (python class signature + JSON example), data sources (parquet paths + column mapping), error modes (404 when season/week missing, 503 when data stale), and the frontend caller (which `matchup-view.tsx` consumer it feeds).

**Endpoint A — Current-week helper** (implements MTCH-04):
- `GET /api/teams/current-week` — returns `{season: int, week: int, source: "schedule" | "fallback"}`.
- Logic: open the latest `data/bronze/schedules/season=YYYY/schedules_*.parquet` (try current calendar year first, then year-1). Find the row where today's date is between `gameday` and `gameday + 6 days`; return its `(season, week)`. If no schedule covers today (offseason), return the max `(season, week)` with `source: "fallback"`.
- Frontend caller: replace `useState(2026)` / `useState(1)` in `MatchupView` with initial state fetched from this endpoint.

**Endpoint B — Team roster with defensive starters** (implements MTCH-02 and MTCH-01 OL):
- `GET /api/teams/{team}/roster?season=YYYY&week=WW&side=offense|defense|all` — returns `TeamRosterResponse`.
- Response: list of `RosterPlayer{player_id, player_name, team, position, depth_chart_position, depth_chart_order, jersey_number, status, snap_pct_offense, snap_pct_defense, injury_status}`.
- Logic:
  - Load latest `data/bronze/players/rosters/season=YYYY/rosters_*.parquet` (if 2026 not present, fall back to max available season and set `fallback: true` in response metadata).
  - Filter to `team == {team}` and `status in ('ACT','RES')` (exclude CUT/RET/DEV/INA for starter view).
  - Left join snap counts from `data/bronze/players/snaps/season=YYYY/week=WW/` on `player_name` (same key the existing `silver_player_transformation` uses) to populate `snap_pct_*`.
  - `side=defense` filter: `position in ('DE','DT','LB','CB','DB','S')` OR `depth_chart_position in ('DE','DT','NT','OLB','ILB','MLB','CB','FS','SS','DB')`. Order by depth_chart_position then by `snap_pct_defense` desc.
  - `side=offense` with OL filter: `depth_chart_position in ('LT','LG','C','RG','RT','T','G')` merged into the response so the frontend can fill the 5 OL slots.
- Frontend caller: `buildDefensiveRoster` becomes `buildDefensiveRosterFromApi(rosterResponse)` — no more `slotHash`. `buildOffensiveRoster` fills OL slots from the same response.

**Endpoint C — Team defensive metrics** (implements MTCH-03):
- `GET /api/teams/{team}/defense-metrics?season=YYYY&week=WW` — returns `TeamDefenseMetricsResponse`.
- Response fields:
  - `overall_def_rating: int` — 1-99 scale derived from `def_sos_rank` percentile (rank 1 of 32 → 99, rank 32 → 50).
  - `def_sos_score: float | null` — raw score from silver SOS.
  - `def_sos_rank: int | null`.
  - `adj_def_epa: float | null` — from silver SOS.
  - `positional: list[PositionalDefenseRank{position: QB|RB|WR|TE, avg_pts_allowed: float, rank: int, rating: int}]` — rating = `101 - round(rank/32 * 49 + 50)` (rank 1 → 99, rank 32 → 50).
  - `source_week: int` — the most recent completed week with data; falls back to season average when the requested week has no data yet.
- Data sources: `data/silver/defense/positional/season=YYYY/opp_rankings_*.parquet` + `data/silver/teams/sos/season=YYYY/sos_*.parquet`.
- Frontend caller: replace the hardcoded `defaults` dict in `buildDefensiveRoster` so defensive slot ratings come from `positional[]`; replace `MatchupAdvantages` diff computation so it also surfaces `positional.rank` for tooltip context.

Also add a **Pydantic schema block** at the bottom showing the exact Python class signatures for `TeamRosterResponse`, `RosterPlayer`, `TeamDefenseMetricsResponse`, `PositionalDefenseRank`, `CurrentWeekResponse` — plans 64-02 and 64-03 will paste these into `web/api/models/schemas.py` unchanged.

Also add a **Fallback matrix** row per endpoint: `requested not present → action` (e.g., rosters 2026 missing → use latest 2025 snapshot + `fallback: true`; defense positional week >= 1 but no data → season-average from max week present).
