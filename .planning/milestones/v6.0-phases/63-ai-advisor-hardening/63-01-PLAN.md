  <name>Task 1: Write scripts/audit_advisor_tools.py</name>
  <files>scripts/audit_advisor_tools.py</files>
  <action>
    Create a single-file Python audit script that probes all 12 advisor tools against the live Railway backend.

    Requirements (all mandatory):
    - Use `httpx` with `timeout=15.0` and a single shared `httpx.Client`. Set `X-API-Key` header from `os.environ.get("RAILWAY_API_KEY", "")` only when non-empty.
    - Read base URL from env `RAILWAY_API_URL`, default to `https://nfldataengineering-production.up.railway.app`.
    - Define a frozen dataclass `ToolProbe` with fields: `tool_name: str`, `endpoint: str`, `params: dict`, `validator: Callable[[dict | list], tuple[bool, str]]`.
    - Define `TOOL_REGISTRY: list[ToolProbe]` with EXACTLY the 12 entries from the interfaces table above. Use the same default params.
    - Per-tool validators (return `(ok, reason)`):
        * getPlayerProjection / compareStartSit / getPositionRankings: body must have key `projections` that is a list with len >= 1; each item must have keys `player_name`, `projected_points`, `position`.
        * searchPlayers: body must be a list with len >= 1; each item must have `player_id`, `player_name`.
        * getNewsFeed / getPlayerNews: body must be a list (len may be 0 in off-season — if 0, mark WARN not FAIL; record reason "empty_payload_offseason").
        * getGamePredictions: body must have `predictions` list; during preseason (week=1, 2026) may be empty — WARN only.
        * getTeamRoster: body must have `lineup` list with len >= 1.
        * getTeamSentiment: body must be a list with len >= 1 and each item must have key `team`. If empty, mark WARN with "no_sentiment_data".
        * getDraftBoard: body must have `board` list with len >= 50 (draft board should always have the full pool).
        * compareExternalRankings: body must have `players` list; each item must have `player_name`, `external_rank`, `our_rank`, `rank_diff`. If `players` is empty — FAIL with "external_source_unavailable".
        * getSentimentSummary: body must have keys `total_articles`, `bullish_players`, `bearish_players`.
    - Implement `probe(client, probe: ToolProbe) -> dict`. Capture for each call: `status_code`, `latency_ms`, `ok` (bool), `reason` (str), `body_keys` (top-level keys or type repr), `sample` (first 200 chars of body), `error` (None or exception repr).
    - Map failure reasons into these categories (string constants):
        BACKEND_DOWN, HTTP_ERROR, EMPTY_PAYLOAD, SCHEMA_MISMATCH, AUTH_BLOCKED, EXTERNAL_SOURCE_DOWN, UNKNOWN.
        Rules:
        - ConnectionError / ReadTimeout → BACKEND_DOWN
        - status_code == 401 → AUTH_BLOCKED
        - status_code >= 500 → HTTP_ERROR
        - status_code == 404 → HTTP_ERROR (route missing)
        - validator returns ok=False and reason contains "external_source" → EXTERNAL_SOURCE_DOWN
        - validator returns ok=False on shape → SCHEMA_MISMATCH
        - validator returns ok=False on empty → EMPTY_PAYLOAD
    - Implement `write_audit_markdown(results: list[dict], out_path: Path) -> None`:
        Write `.planning/phases/63-ai-advisor-hardening/TOOL-AUDIT.md` with:
        - H1 "# Advisor Tool Audit — Baseline"
        - Metadata block (run timestamp UTC, base URL, auth header present yes/no, total PASS / WARN / FAIL counts)
        - A markdown table with columns: Tool | Endpoint | HTTP | Latency (ms) | Status | Reason | Sample
        - A second section "## Failure Detail" listing each FAIL with the full error + first 500 chars of body.
    - `main()` runs all 12 probes, prints a one-line summary to stdout (e.g. `AUDIT: 9 PASS / 2 WARN / 1 FAIL`), writes the markdown, and exits with code 0 if total FAIL == 0 else 1.
    - Type hints on every function, Google-style docstrings on top-level functions, use `logging` (not print) for non-summary output. Follow the Python rules in .claude/rules/coding-style.md.
    - NO secrets embedded in source. Read from env only.
    - Add `"httpx"` to requirements if not already present (check requirements.txt first — do not add if present).
  </action>
