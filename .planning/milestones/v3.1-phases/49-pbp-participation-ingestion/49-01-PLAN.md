- Run `bronze_ingestion_simple.py` with `--include-participation` for seasons 2020-2025
- Store output in `data/bronze/pbp_participation/season=YYYY/` as timestamped Parquet files
- Verify each season file contains `game_id`, `play_id`, `offense_players`, `defense_players` columns
- Validate row counts: expect ~45K-55K plays per season (regular season + playoffs)

### T-02: Extend to 2016-2019 for Full Training History
- Run same ingestion for seasons 2016-2019
- Validate data availability (nfl-data-py coverage starts at 2016)

- Check play coverage: % of PBP plays that have corresponding participation records
- Validate no duplicate play_id within a season
- Confirm offense_players contains exactly 11 player IDs per snap (where available)
- Verify idempotency: re-running ingestion does not create duplicate records

### T-04: Error Handling
- Handle nfl-data-py API failures gracefully (retry with backoff)
- Log seasons/weeks with missing participation data
- Ensure partial failures don't corrupt previously-ingested seasons

- nfl-data-py library with participation data support
