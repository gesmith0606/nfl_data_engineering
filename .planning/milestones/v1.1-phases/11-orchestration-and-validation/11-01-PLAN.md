  <action>
    Write tests first (RED), then implement the batch script (GREEN).

    **Tests** (`tests/test_batch_ingestion.py`):
    - Mock `NFLDataAdapter` to return small DataFrames (5 rows) for most types and raise Exception for one type
    - Mock `save_local` to be a no-op (no disk writes in tests)
    - Use `@patch.object` pattern consistent with existing test suite
    - Test `run_batch()` function directly (not CLI)
    - Test `already_ingested()` helper with temp directory containing fake parquet files

    **Script** (`scripts/bronze_batch_ingestion.py`):
    - CLI with argparse: `--season-start` (default 2016), `--season-end` (default 2025), `--force` (skip-existing override), `--dry-run` (show what would run)
    - Core function `run_batch(season_start, season_end, skip_existing, dry_run) -> List[Result]` where `Result = Tuple[str, Optional[str], int, str, str]` (data_type, variant, season, status, detail)
    - `already_ingested(data_type, entry, season, variant, base_dir)` checks for existing parquet files matching the bronze_path pattern
    - Loop structure: iterate `DATA_TYPE_REGISTRY.items()`, determine variants (sub_types or QBR frequencies or single pass), determine valid seasons from `DATA_TYPE_SEASON_RANGES`, then per-season: check skip-existing, fetch via adapter method, validate, save via `save_local`, handle week_partition
    - For PBP: add `gc.collect()` after each season to prevent memory accumulation (per research pitfall)
    - For 0-row returns: record as ("SKIP", "0 rows returned") not as failure
    - Wrap each type/variant/season in try/except: on failure record ("FAIL", str(e)) and continue
    - After loop: print summary table with counts of OK/SKIP/FAIL grouped by data type
    - Use `_build_method_kwargs` approach but build kwargs inline (avoid importing argparse Namespace dependency). Instead, construct kwargs directly from registry entry: seasons=[season], plus sub_type/stat_type/frequency for variant types, plus PBP_COLUMNS/downcast/include_participation for PBP
    - Validation: call `adapter.validate_data(df, data_type)` and `format_validation_output()` for each non-empty df, wrapped in try/except (warn-never-block pattern from Phase 6)
    - Progress output format: `[{idx}/{total_types}] {data_type}` header, then per-season `  Season {year}... {status} ({rows} rows, {cols} cols)`
    - Exit code: 0 if no failures, 1 if any failures

    **Important:** Do NOT import `_build_method_kwargs` from bronze_ingestion_simple.py -- it depends on argparse Namespace. Build kwargs directly from registry entries to keep the batch script self-contained except for DATA_TYPE_REGISTRY and save_local.
