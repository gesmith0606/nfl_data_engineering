# Sketch 003: Rankings Data Density

## Design Question
The home is a story page; Rankings is the workhorse users live in. This sketch
decides how the broadcast identity (condensed caps, mint/yellow, near-black)
translates to a dense, functional data surface — the pattern every interior data
page (Scores, Matchups, My League) will copy.

## How to View
open .planning/sketches/003-rankings-data-density/index.html

All three variants share ONE live data set and ONE control bar — position tabs,
PPR/Half/Std scoring toggle (actually re-computes projections), and search all
work in every variant.

## Variants
- **A: Broadcast Table** — classic dense table: yellow condensed headers under a
  mint rule, zebra rows, mint proj numerals, sortable Proj column, click a row
  to expand the conformal floor/ceiling band + stack correlation chip.
- **B: Tier Board** — rows grouped under yellow tier headlines ("Tier 1 — The
  Elite") with fading gold rules; airier apple-style rows, proj right-aligned.
- **C: Pill Rows** — every player is a scorebug-style pill: dark capsule, mint
  proj panel on the right, multicolor gradient outline reserved for Tier 1s.

## What to Look For
- **Density vs identity:** A holds the most data per screen — does it still feel
  broadcast, or just like a dark table? C is the most branded — does it get
  exhausting after 15 rows? B splits the difference.
- Work the controls in each variant: switch to WR, toggle PPR (watch WR/TE values
  jump vs QBs), search "det" for the Lions stack.
- In A, click Amon-Ra St. Brown — the expanded row shows the conformal band and
  the Goff +0.51 stack chip. Is that the right reveal pattern for player depth?
- Check 375px with the 📱 toolbar button — which treatment degrades best?
