# Sketch 005: Scores Grid

## Design Question
The compact scorebug has only appeared 3-up in a marketing row. A real Scores page
must handle a 10-16 game slate with hierarchy (which games do we disagree with the
market on?) and states (upcoming predictions vs graded finals). The graded state IS
the receipts story — misses stay visible.

## How to View
open .planning/sketches/005-scores-grid/index.html

**Toggle the week chips top-right**: "2026 · WK 1" shows upcoming predictions with
edge tags; "2025 · WK 18 ✓ graded" switches every variant to graded finals — ✓ COVER
bugs keep the mint outline, ✗ MISS bugs go gray (but stay on the board), and an
ATS record banner (10–5–1, +4.2u) appears.

## Variants
- **A: Edge Board** — high-edge games render as enlarged hero bugs under a yellow
  "High edge" kicker; the rest of the slate sits in a compact grid below. Sorted
  by conviction, not kickoff.
- **B: Broadcast Slate** — a TV-style slate grouped by time slot (SUN 1:00 / 4:05 /
  4:25 / SNF / MNF), uniform compact bugs, edge flagged via corner tags only.
- **C: Ledger** — the 003 table pattern applied to games: our proj score, our line
  vs market line side-by-side, edge level, cover prob (or graded result). Densest
  and most audit-like.

## What to Look For
- Hierarchy: does conviction-first (A) beat schedule-first (B)? Does the ledger (C)
  read as "receipts" or just a spreadsheet?
- Toggle to the graded week in each variant — where does the ✓/✗ story land hardest?
- The EDGE/LEAN corner tags vs a dedicated edge column (C) — which communicates
  "we disagree with Vegas here" faster?
