The pipeline default changes so that rule-based extraction always runs first (the current auto-mode already falls back to rules when no key is set — we strengthen this to "rules primary, LLM never drives model-facing fields").

New module `src/sentiment/enrichment/llm_enrichment.py` API:
```python
class LLMEnrichment:
    def __init__(self, model: str = "claude-haiku-4-5") -> None: ...
    @property
    def is_available(self) -> bool: ...
    def enrich(self, silver_record: Dict[str, Any]) -> Dict[str, Any]:
        """
        Takes a Silver record (already carrying rule-extracted events) and returns
        a copy with two NEW fields added:
            - summary: str  (single sentence, website-facing)
            - refined_category: str (may override the rule-extracted category if LLM is confident)

        The ORIGINAL event flags are never overwritten. If the LLM call fails or
        the key is missing, the record is returned unchanged.
        """

def enrich_silver_records(
    season: int,
    week: int,
    dry_run: bool = False,
) -> int:
    """
    Walk Silver signal JSON files for (season, week), call LLMEnrichment.enrich()
    on each record, and write enriched sidecar files to
    data/silver/sentiment/signals_enriched/season=YYYY/week=WW/.

    Non-destructive: original signals files are untouched.
    Returns: number of records enriched (0 if disabled).
    """
```

News service (`web/api/services/news_service.py` — modified in Plan 05) must be updated in this plan to OPTIONALLY read the `signals_enriched/` sidecar first, merge the `summary` and `refined_category` fields into returned items, and fall back silently to non-enriched records.

`NewsItem.summary` Pydantic field already added in Plan 05; this plan populates it when data is available.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-61-06-01 | Information Disclosure | API key in logs | mitigate | Never log ANTHROPIC_API_KEY value; only log `bool(os.environ.get("ANTHROPIC_API_KEY"))` |
| T-61-06-02 | Tampering | LLM returns prompt injection | mitigate | summary clamped to 200 chars; refined_category validated against allow-list; React auto-escapes rendered text in Plan 05 components |
| T-61-06-03 | Denial of Service | Anthropic API rate limits | mitigate | enrich() returns record unchanged on any exception; pipeline step success=True even when LLM fails (fail-open per D-06) |
| T-61-06-04 | Repudiation | Silent enrichment drift | mitigate | Every pipeline run logs enrichment enabled/disabled state and record count |
| T-61-06-05 | Elevation of Privilege | LLM overwriting event flags | mitigate | enrich() only ADDS `summary` and `refined_category` fields; rule-extracted event flags are never modified |
