**Structure:**

```python
#!/usr/bin/env python3
"""
Silver Layer - Game Context Transformation Script

Reads schedules data from local Bronze parquet files, computes game context
features (weather, rest/travel, coaching), and writes to the local Silver
layer. Optionally uploads to S3 if AWS credentials are available.

Usage:
    python scripts/silver_game_context_transformation.py --season 2024
    python scripts/silver_game_context_transformation.py --seasons 2016 2017 2018 2019 2020 2021 2022 2023 2024 2025
"""
```

**Imports:**
```python
import sys, os, argparse, glob as globmod

from game_context import compute_game_context
```

**Helper functions** (copy from silver_team_transformation.py, adapt for schedules):

`_read_local_schedules(season: int) -> pd.DataFrame`:
- Pattern: `data/bronze/schedules/season={season}/*.parquet`
- Read latest file (sorted, take last)
- Return DataFrame or empty DataFrame if not found

`_save_local_silver(df, key, ts)` and `_try_s3_upload(df, bucket, key)`:
- Copy exactly from silver_team_transformation.py (same implementation)

**Main transform function:**

`run_game_context_transform(seasons: list, s3_bucket: Optional[str] = None) -> None`:
- Sort seasons ascending (critical for coaching: need prior season context)
- Track `prior_season_df = None` across loop iterations
- For each season:
  1. Read schedules from local Bronze via `_read_local_schedules(season)`
  2. If empty, print warning, skip
  3. Call `compute_game_context(schedules_df, prior_season_df)`
  4. Print stats: row count, team count, column count
  5. Save to Silver: `SILVER_TEAM_S3_KEYS["game_context"].format(season=season, ts=ts)`
  6. Optionally upload to S3
  7. Set `prior_season_df = schedules_df` for next iteration

**CLI** (main function):
- argparse with `--season`, `--seasons`, `--no-s3` flags
- Default season: 2024
- S3 logic: same pattern as silver_team_transformation.py
- Print header: "NFL Silver Layer - Game Context Transformation"
