**2. Create src/game_context.py** with these functions:

```python
"""
Game Context Module — Schedule-derived features for prediction models.

Extracts weather, rest/travel, and coaching features from schedules Bronze data.
Unpivots home/away game rows into per-team per-week rows.
All output joinable on [team, season, week].
"""
```

Functions to implement (in this order):

**`_haversine_miles(lat1, lon1, lat2, lon2) -> float`**
- Standard haversine using `math` module (radians, sin, cos, sqrt, atan2)
- Earth radius = 3958.8 miles
- Returns great-circle distance in miles

**`_timezone_diff_hours(tz_name1, tz_name2, game_date) -> float`**
- `game_date` is string "YYYY-MM-DD"
- Use `pytz.timezone()` and `.localize(datetime(... hour=12))` for DST-aware offsets
- Return `abs(offset1 - offset2)` in hours
- Handle missing/invalid tz names by returning 0.0

**`_unpivot_schedules(schedules_df) -> pd.DataFrame`**
- Create `home` df: rename home_team->team, away_team->opponent, home_coach->head_coach, away_coach->opponent_coach, home_rest->rest_days, away_rest->opponent_rest; assign is_home=True
- Create `away` df: rename away_team->team, home_team->opponent, away_coach->head_coach, home_coach->opponent_coach, away_rest->rest_days, home_rest->opponent_rest; assign is_home=False
- Select columns: game_id, season, week, team, opponent, head_coach, opponent_coach, rest_days, opponent_rest, is_home, temp, wind, roof, surface, stadium_id, stadium, game_type, gameday, location
- Concat, sort by [team, season, week], reset_index(drop=True)
- Return DataFrame

**`compute_weather_features(unpivoted_df) -> pd.DataFrame`**
- Start with team, season, week columns
- `is_dome`: `roof.isin(['dome', 'closed'])`
- `temperature`: copy from `temp`; set to 72.0 where is_dome is True
- `wind_speed`: copy from `wind`; set to 0.0 where is_dome is True
- `is_high_wind`: `(wind_speed > 15).fillna(False)` — cast to bool
- `is_cold`: `(temperature <= 32).fillna(False)` — cast to bool
- `surface`: pass through from unpivoted_df
- Return df with columns: team, season, week, is_dome, temperature, wind_speed, is_high_wind, is_cold, surface

**`compute_rest_features(unpivoted_df) -> pd.DataFrame`**
- Start with team, season, week columns
- `rest_days`: `unpivoted_df['rest_days'].clip(upper=14)` — cap at 14
- `opponent_rest`: `unpivoted_df['opponent_rest'].clip(upper=14)`
- `is_short_rest`: `rest_days <= 6` — bool
- `is_post_bye`: `rest_days >= 13` — bool
- `rest_advantage`: `rest_days - opponent_rest` — int/float
- Return df with columns: team, season, week, rest_days, opponent_rest, is_short_rest, is_post_bye, rest_advantage

**`compute_travel_features(unpivoted_df) -> pd.DataFrame`**
- Start with team, season, week columns
- For each row, look up team's home stadium coords from `STADIUM_COORDINATES` (using team abbreviation -> lat, lon)
- Look up game venue coords from `STADIUM_ID_COORDS` (using stadium_id -> lat, lon, tz)
- `travel_miles`: If is_home is True AND location != 'Neutral', set to 0.0. Otherwise, haversine from team's home to game venue.
- `tz_diff`: `_timezone_diff_hours(team_home_tz, venue_tz, gameday)` — absolute hours
- Handle missing stadium_id mappings: log warning, set travel_miles=NaN, tz_diff=0.0
- Return df with columns: team, season, week, travel_miles, tz_diff

**`compute_coaching_features(unpivoted_df, prior_season_coaches=None) -> pd.DataFrame`**
- `prior_season_coaches`: Optional dict of `{team: coach_name}` from prior season's final week. If None (first season = 2016), all coaching_change = False.
- Start with team, season, week, head_coach columns
- Off-season change: For Week 1 (or first week of season in data), compare head_coach to prior_season_coaches[team]. If different, coaching_change=True.
- Mid-season change: For each team sorted by week, compare head_coach to previous week's head_coach. If different, coaching_change=True.
- `coaching_change`: bool flag set True from the first week of a new coach onward through end of season (NOT just the transition week — set True for the transition week, then False for subsequent weeks with same coach)
- Actually, re-reading CONTEXT.md: "set `coaching_change=True` for Week N+1 onward" for mid-season, and "set `coaching_change=True` for all weeks of new season" for off-season. So coaching_change stays True for ALL remaining weeks after a change within that detection scope.
- Wait — re-reading more carefully: mid-season = True from change week onward; off-season = True for all weeks of the new season. So: `coaching_change` = True if the head coach at any point in the current season differs from who started.
- Simplest correct implementation: For each (team, season), mark coaching_change=True for all rows if (a) Week 1 coach differs from prior season's final coach, OR (b) coach changes mid-season at any point. For mid-season, only mark from the change week onward.
- `coaching_tenure`: consecutive weeks with same coach. For Week 1 returning coaches, carry over from prior season (prior_tenure + weeks_gap). For new coaches, start at 1. Increment by 1 each subsequent week with same coach. Reset to 1 on mid-season change.
- Return df with columns: team, season, week, head_coach, coaching_change, coaching_tenure

**`compute_game_context(schedules_df, prior_season_df=None) -> pd.DataFrame`**
- Orchestrator: calls `_unpivot_schedules()`, then all four compute functions
- `prior_season_df`: schedules for prior season (used to extract prior_season_coaches dict)
- If prior_season_df provided, extract last-week coaches: unpivot prior season, group by team, take max week's head_coach
- Merge all results on [team, season, week] using left join
- Include is_home and game_type from unpivoted base
- Return combined DataFrame
