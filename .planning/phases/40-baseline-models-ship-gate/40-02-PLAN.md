<objective>
Build the ship gate evaluation CLI that trains all 19 per-position stat models, re-runs the heuristic baseline on identical rows, compares MAE per position, and produces a clear SHIP/SKIP verdict per position.

Purpose: This is the decision gate that determines which positions get ML predictions vs heuristic fallback. It ensures fair comparison (identical rows), dual agreement (OOF + holdout), and a safety floor (no stat >10% worse).

Output: `scripts/train_player_models.py` CLI script that trains models and produces the ship gate report. `tests/test_player_ship_gate.py` with tests for verdict logic.
</objective>

    ```python
    def ship_gate_verdict(position, ml_mae, heuristic_mae, oof_ml_mae, oof_heuristic_mae, per_stat_results):
        """Per-position ship-or-skip with dual agreement and safety floor."""
        holdout_improvement = (heuristic_mae - ml_mae) / heuristic_mae if heuristic_mae > 0 else 0.0
        oof_improvement = (oof_heuristic_mae - oof_ml_mae) / oof_heuristic_mae if oof_heuristic_mae > 0 else 0.0

        # D-09: Safety floor — no individual stat >10% worse
        safety_violation = False
        for stat_result in per_stat_results:
            if stat_result["ml_mae"] > stat_result["heuristic_mae"] * 1.10:
                safety_violation = True
                break

        # D-08: 4%+ improvement required
        # D-10: Dual agreement — both OOF and holdout must pass
        ship = (holdout_improvement >= 0.04
                and oof_improvement >= 0.04
                and not safety_violation)

        return {
            "position": position,
            "ml_mae": round(ml_mae, 4),
            "heuristic_mae": round(heuristic_mae, 4),
            "holdout_improvement_pct": round(holdout_improvement * 100, 2),
            "oof_improvement_pct": round(oof_improvement * 100, 2),
            "safety_violation": safety_violation,
            "verdict": "SHIP" if ship else "SKIP",
        }
    ```

       - `--holdout-eval` (flag) — evaluate on 2025 holdout after training (default: False)

    2. **Main flow:**
       ```
       a. Load data: assemble_multiyear_player_features(PLAYER_DATA_SEASONS)
       b. Get feature columns: get_player_feature_columns(all_data)
       c. Run leakage check: detect_leakage(all_data, feature_cols, PLAYER_LABEL_COLUMNS)
       d. Feature selection: run_player_feature_selection(all_data, feature_cols, positions)

       e. For each position:
          i.   Filter to position data
          ii.  Train all stat models: train_position_models(pos_data, position, feature_cols_by_group)
          iii. Generate ML predictions on OOF data using predict_player_stats

          vii. Per-stat MAE comparison for safety floor check
          viii. Run ship_gate_verdict(position, ml_mae, heuristic_mae, oof_ml_mae, oof_heuristic_mae, per_stat_results)
       f. Build and save report: build_ship_gate_report(position_results)

       ```
