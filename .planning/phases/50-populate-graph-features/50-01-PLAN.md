# Phase 50-01: Populate Graph Features from Participation Data - Plan

**Created:** 2026-04-02
**Status:** Complete

## Tasks

### T-01: Compute WR Matchup Features
- WR-vs-defense EPA from participation co-occurrence
- WR-CB co-occurrence quality (snap counts where WR targeted while CB on field)
- Similar-WR-vs-defense graph traversal
- Cache as `data/silver/graph_features/season=YYYY/graph_wr_matchup_TIMESTAMP.parquet`

### T-02: Compute OL/RB Features
- OL starter count and backup insertion features from participation data
- OL continuity score (rolling % of snaps with same 5 starters)
- RB YPC with full OL vs backup OL delta
- Cache as `data/silver/graph_features/season=YYYY/graph_ol_rb_TIMESTAMP.parquet`

### T-03: Compute TE Matchup Features
- TE-LB/Safety coverage rate from participation data
- Opposing defense fantasy points allowed to TEs (rolling 3 games)
- TE red zone target share redistribution
- Cache as `data/silver/graph_features/season=YYYY/graph_te_matchup_TIMESTAMP.parquet`

### T-04: Compute Scheme Features
- Defensive front quality vs run
- Scheme matchup scoring (team run scheme type vs opposing defense front)
- Gap-specific YPC vs defense
- Cache as `data/silver/graph_features/season=YYYY/graph_scheme_TIMESTAMP.parquet`

### T-05: Combined Feature Assembly
- Merge all feature groups into `graph_all_features_TIMESTAMP.parquet` per season
- Enforce temporal lag (shift(1), no future data leakage)
- Validate coverage targets: 60%+ non-null for WR, RB, TE features
- Fix _join_graph_features glob pattern to avoid picking up wrong file types

## Dependencies
- Phase 49 (PBP participation data must exist in Bronze)
- Bronze PBP, rosters, depth charts, injuries, player_weekly, PFR defensive, schedules

## Success Criteria
1. WR features populated with 60%+ non-null values for WR player-weeks
2. RB/OL features populated with 60%+ non-null values for RB player-weeks
3. TE features populated with 60%+ non-null values for TE player-weeks
4. All features respect temporal lag (shift(1), no future data)
5. Features cached in `data/silver/graph_features/` per season
