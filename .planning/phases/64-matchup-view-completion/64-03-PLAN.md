<objective>
Expose real team defensive metrics via a FastAPI endpoint that the matchup view will consume. This is the MTCH-03 backend — "matchup advantage indicators calculated from real team defensive metrics, not hardcoded."

Purpose: The existing matchup view computes defensive ratings from `slotHash(team, slot)` which is deterministic noise. This plan replaces that entire compute path with values derived from `silver/defense/positional` (points allowed by position) and `silver/teams/sos` (adjusted EPA-based SOS). The frontend in plan 64-04 will compute offense-vs-defense advantage by comparing a projected-player's percentile against the opposing team's positional rank.

This plan runs sequentially AFTER 64-02 (wave 3, depends_on [64-01, 64-02]) because both plans append to `web/api/models/schemas.py` and `web/api/main.py`. Running them sequentially eliminates merge conflicts. The defense router still lives in a separate file (`teams_defense.py`) so there is no overlap with 64-02's `teams.py`. `main.py` edits are additive-only (new import + new include_router call) — apply them cleanly on top of 64-02's additions.

Output:
- `/api/teams/{team}/defense-metrics?season&week` — returns overall + per-position defensive ratings from real silver data.
- Pytest suite validating rating bounds, positional coverage, and season-average fallback.
</objective>
