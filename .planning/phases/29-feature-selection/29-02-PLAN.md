       - Log each count's MAE for visibility

    2. `run_final_selection(all_data, feature_cols, target_col, optimal_count, correlation_threshold=0.90, params=None)` function:
       - Run `select_features_for_fold()` on ALL training data (seasons < HOLDOUT_SEASON) at optimal_count
       - This produces the definitive SELECTED_FEATURES list (per Research open question 2 resolution)
       - Return the FeatureSelectionResult

    3. `save_metadata(result, cutoff_results, output_dir)` function:
       - Create `models/feature_selection/` directory if needed

         - `selected_features`: list of feature names
         - `dropped_correlation`: dict of dropped -> kept partner
         - `dropped_low_importance`: list of low-importance features
         - `shap_scores`: dict of feature -> mean |SHAP|
         - `correlated_pairs`: list of [feat_a, feat_b, r] triples
         - `optimal_cutoff`: the best count
         - `cv_mae_by_cutoff`: dict of count -> mean MAE
         - `n_original`, `n_after_correlation`, `n_selected`
         - `fold_seasons`: seasons used for final selection
         - `timestamp`: ISO datetime string

    4. `main(argv=None)` CLI entry point:
       - argparse with:
         - `--target` choices=["spread", "total"] default="spread"
         - `--counts` nargs="+" type=int default=[60, 80, 100, 120, 150]
         - `--correlation-threshold` type=float default=0.90
         - `--dry-run` flag (find optimal count but don't update config.py)
       - Load data: `assemble_multiyear_features(TRAINING_SEASONS)` then `get_feature_columns()`
       - Filter to training seasons only (exclude HOLDOUT_SEASON rows) -- FSEL-04

       - Call `run_final_selection()` with optimal count
       - Call `save_metadata()`
       - Unless `--dry-run`, write SELECTED_FEATURES to config.py (Task 2 handles the config update)
       - Print summary: optimal count, MAE at each count, number dropped by correlation vs importance
