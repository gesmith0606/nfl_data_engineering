# Phase 54-01: Unified Evaluation Pipeline — Plan

**Created:** 2026-04-03
**Status:** Ready

## Objective

Align residual model training and backtest to use identical production heuristic + full 466-feature set, then retrain and re-evaluate all positions.

## Tasks

### T-01: Extract Production Heuristic as Standalone Function
- Create `src/unified_evaluation.py`
- Extract `compute_production_heuristic(player_weekly_df, schedules_df, season, week, scoring_format)` that replicates the exact production heuristic:
  - `_weighted_baseline()` with RECENCY_WEIGHTS (roll3=0.30, roll6=0.15, std=0.55)
  - Usage multiplier [0.80-1.15]
  - Matchup multiplier [0.85-1.15]
  - Vegas multiplier (implied_total / 23.0)
  - `_apply_ceiling_shrinkage()` at 12/18/23 thresholds
  - NO injury adjustments (applied separately)
- Must be callable from both `hybrid_projection.py` (training) and `backtest_projections.py` (evaluation)
- Return DataFrame with player_id, season, week, heuristic_projected_points

### T-02: Full Feature Assembly in Backtest
- Update `scripts/backtest_projections.py` to add `--full-features` flag
- When enabled, call `assemble_player_features(season)` per backtest season
- Join features to player-week rows during evaluation
- This provides the full 466-column feature vector instead of 42 Silver columns
- Feature assembly must respect temporal lag (shift(1), only prior data)

### T-03: Retrain Residual Models Against Production Heuristic
- Update `src/hybrid_projection.py` `train_and_save_residual_models()`:
  - Use `compute_production_heuristic()` from unified_evaluation.py (not simplified version)
  - Use full 466-feature set from `assemble_player_features()`
  - Train for ALL positions (QB, RB, WR, TE) — not just WR/TE
- Retrain and save to `models/residual/`

### T-04: Run Full-Feature Backtest
- Run backtest with full features:
  ```
  python scripts/backtest_projections.py --seasons 2022,2023,2024 --scoring half_ppr --ml --full-features
  ```
- Report per-position MAE comparison:
  - Heuristic standalone
  - Current hybrid (42 features)
  - New hybrid (466 features)
- Ship/skip per position based on improvement

### T-05: Update Router If Needed
- If QB or RB now benefit from residual correction, update ML projection router
- If WR/TE improve further with full features, update model artifacts

### T-06: Tests
- Test `compute_production_heuristic()` matches projection_engine output
- Test full-feature assembly during backtest
- Test residual model with production heuristic alignment
- Run full suite
