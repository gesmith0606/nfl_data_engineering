def apply_team_rolling(
    df: pd.DataFrame,
    stat_cols: List[str],
    windows: Optional[List[int]] = None,
) -> pd.DataFrame:
    """Apply shifted rolling averages and season-to-date expanding average by team.
    For each stat column and window size, creates {col}_roll{window} using
    shift(1).rolling(window, min_periods=1).mean() grouped by ['team', 'season'].
    Also creates {col}_std (season-to-date expanding average).
    The shift(1) ensures only prior weeks are used (no look-ahead bias)."""
```

"""Silver Player Quality Transformation.
Aggregates Bronze player data (weekly stats, depth charts, injuries)
to team-level per-week player quality features.
Output: data/silver/teams/player_quality/season=YYYY/
"""
import argparse, glob, os, sys
import numpy as np
import pandas as pd

PROJECT_ROOT = os.path.join(os.path.dirname(__file__), "..")
sys.path.insert(0, os.path.join(PROJECT_ROOT, "src"))

from team_analytics import apply_team_rolling

BRONZE_DIR = os.path.join(PROJECT_ROOT, "data", "bronze")
SILVER_DIR = os.path.join(PROJECT_ROOT, "data", "silver")

INJURY_MULTIPLIERS = {
    "Active": 1.0, "Questionable": 0.85, "Doubtful": 0.50,
    "Out": 0.0, "IR": 0.0, "PUP": 0.0,
}
```

Functions to implement:
- `_read_local_bronze(data_type, season)` -- read latest parquet from `BRONZE_DIR/{data_type}/season={season}/`
- `compute_qb_quality(weekly_df, depth_df)` -- Per D-01, D-02, D-03, D-04:
  - From depth_df: filter `pos_abb=='QB'` and `depth_team=='1'`, rename `club_code` to `team`
  - From weekly_df: filter position=='QB', find max-attempts QB per [recent_team, season, week], rename recent_team to team
  - Merge both signals. Set `backup_qb_start = (depth_starter_id != actual_starter_id)`
  - Output: [team, season, week, qb_passing_epa, backup_qb_start]
- `compute_positional_quality(weekly_df)` -- Per D-05, D-06, D-08:
  - Compute carry_share = carries / team_total_carries (per D-08 research: this column does NOT exist in Bronze)
  - Top 2 RBs by carries per [team, week]: rb_weighted_epa = sum(rushing_epa * carry_share) / sum(carry_share)
  - Top 3 WR/TEs by targets per [team, week]: wr_te_weighted_epa = sum(receiving_epa * target_share) / sum(target_share)
  - Output: [team, season, week, rb_weighted_epa, wr_te_weighted_epa]
- `compute_injury_impact(injury_df, weekly_df)` -- Per D-09, D-10, D-11:
  - Map report_status to INJURY_MULTIPLIERS (default Active=1.0 for missing)
  - For each position group (QB / RB+WR+TE / defensive positions):
    - Join with weekly_df to get usage shares (target_share for WR/TE, carry_share for RB)
    - impact = sum(usage_share * (1 - multiplier)) per team per week
  - Output: [team, season, week, qb_injury_impact, skill_injury_impact, def_injury_impact]
- `transform_season(season)` -- orchestrate all three, merge on [team, season, week], call `apply_team_rolling()` with stat_cols = [qb_passing_epa, rb_weighted_epa, wr_te_weighted_epa, qb_injury_impact, skill_injury_impact, def_injury_impact], windows=[3, 6]
- `_save_local_silver(df, key)` -- write parquet to SILVER_DIR/{key}
- `main()` with argparse: `--seasons` (list of ints), e.g., `python scripts/silver_player_quality_transformation.py --seasons 2020 2021 2022 2023 2024`

Output columns after apply_team_rolling (per D-02, D-06):
- qb_passing_epa, qb_passing_epa_roll3, qb_passing_epa_roll6, qb_passing_epa_std
- rb_weighted_epa, rb_weighted_epa_roll3, rb_weighted_epa_roll6, rb_weighted_epa_std
- wr_te_weighted_epa, wr_te_weighted_epa_roll3, wr_te_weighted_epa_roll6, wr_te_weighted_epa_std
- qb_injury_impact, qb_injury_impact_roll3, qb_injury_impact_roll6, qb_injury_impact_std
- skill_injury_impact, skill_injury_impact_roll3, skill_injury_impact_roll6, skill_injury_impact_std
- def_injury_impact, def_injury_impact_roll3, def_injury_impact_roll6, def_injury_impact_std
- backup_qb_start (boolean, no rolling needed)

That is 6 raw + 18 rolling + 1 boolean = 25 new columns per team per week. After home/away split and differential computation in feature_engineering.py, this yields approximately 25-30 new game-level features (rolling columns get diff_ prefix, backup_qb_start gets _home/_away suffixes).

Handle early-season NaN: apply_team_rolling uses min_periods=1, so week 2 (first after shift(1)) will have values. Week 1 will be NaN for all rolling features -- this is correct behavior.

Handle missing Bronze data: if any Bronze source is empty for a season, log a warning and return an empty DataFrame for that source's contribution. The merge uses pd.merge how='left' so missing data produces NaN columns.
