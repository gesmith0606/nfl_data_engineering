| Claude Haiku extraction → Bronze data | Read-only on Bronze; writes new Silver parquet files (append-only, date-partitioned) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-69-01 | Information disclosure | ANTHROPIC_API_KEY logged in workflow output | mitigate | `process_sentiment.py` uses `os.environ.get()` directly; never echoes value. Workflow uses `env:` block with `${{ secrets.ANTHROPIC_API_KEY }}` (GitHub auto-masks secret values in logs). |
| T-69-02 | Denial of service | Anthropic API rate-limit from unbounded extraction | mitigate | Scope is 2 weeks (~18 Bronze source files × small player-week fan-out). Well below rate limits. `processed_ids.json` prevents re-extraction of already-processed docs. |
| T-69-03 | Tampering | Malicious Bronze source poisoning sentiment | accept | Bronze sources (RSS + Sleeper + Reddit + RotoWire + PFT) are already ingested upstream. Out of scope for this phase. Downstream: Claude Haiku extraction is deterministic within prompt; no prompt injection surface beyond article text (handled by Phase 61's escape strategy). |
| T-69-04 | Repudiation | Backfill run untraceable | mitigate | GHA logs retained 90 days; every run committed to main with `github-actions[bot]` identity; `roster_changes.log` pattern from Phase 67 adopted by extractor. |
