# Phase 57-01: Quantile Regression — Plan

**Created:** 2026-04-03

## Tasks

### T-01: Quantile Model Training Module
Create `src/quantile_models.py`:
- `train_quantile_models(features_df, targets_df, positions, quantiles=[0.1, 0.5, 0.9])`:
  - For each position, train 3 LightGBM models (10th, 50th, 90th percentile)
  - Use walk-forward CV (train 2016-N, validate N+1)
  - Features: 42-column Silver set (same as hybrid residual)
  - Target: total fantasy points (not per-stat)
  - Return: dict of trained models + OOF predictions per quantile
- `save_quantile_models(models, path='models/quantile/')` 
- `load_quantile_models(path='models/quantile/')`
- `predict_quantiles(models, features_df)` → DataFrame with floor, projection, ceiling columns

### T-02: Training CLI
Create `scripts/train_quantile_models.py`:
- CLI: `python scripts/train_quantile_models.py --scoring half_ppr`
- Train for all 4 positions
- Save models to `models/quantile/`
- Report OOF MAE for 50th percentile vs heuristic
- Report calibration: % of actuals within 10th-90th range

### T-03: Calibration Evaluation
- For each position, compute: P(10th < actual < 90th)
- Target: 80% coverage
- Also compute: P(actual < 10th) and P(actual > 90th) — should be ~10% each
- If miscalibrated, adjust alpha values (e.g., 0.05/0.95 for wider intervals)

### T-04: Wire into Projection Engine
Update `src/projection_engine.py`:
- Replace `add_floor_ceiling()` hardcoded variance with quantile predictions
- When quantile models are available: floor = 10th percentile, ceiling = 90th percentile
- When unavailable: fall back to current variance-based floor/ceiling
- Point estimate remains heuristic (or hybrid for WR/TE) — 50th percentile is supplementary

### T-05: Update Web API

- Ensure projected_floor and projected_ceiling in API response come from quantile models
