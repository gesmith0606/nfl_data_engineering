This plan consumes the `## Priority Targets for Phase 62 Plans > DSGN-04` list in AUDIT-BASELINE.md. That list enumerates, per page, the specific mobile-breaking issues (overflow, touch-target failures, broken grids).

### Breakpoint strategy (consistent with Tailwind v4 defaults)
- Base (default): mobile, ~375px. Design primary.
- `md:` (≥768px): tablet/desktop.
- `lg:` (≥1024px): wide desktop.
Everything under `md:` must work at 375px.

### Data-table adaptation rules

For projections-table and rankings-table (both data-dense):
- EITHER: wrap in an overflow-x-auto container with sticky first column (player name) and explicit min-width on the table, AND show a small "scroll →" affordance on first render at mobile width.
- OR: render a card variant below `md:` where each row becomes a vertically-stacked card with the key stats labeled.
- Pick whichever the design-engineer agent judges higher-quality per AUDIT-BASELINE findings. Document the choice in the SUMMARY.

### Tappable-target rule
Any element with `onClick`, `role='button'`, `<button>`, `<a>` must be ≥ 44×44 CSS px at mobile viewport. Use `min-h-11 min-w-11` (44px with 4px spacing base) or explicit `--space-11` token if added.

### Sidebar on mobile
The existing shadcn sidebar likely already has a mobile drawer mode. Verify it. If not functional, wire it through (the shadcn sidebar primitive exposes `isMobile` + `<Sheet>` integration).

### Guardrails
- Do not regress desktop layouts. After changes, desktop (≥ md) must look identical to post-62-04 state.
- Do not add new dependencies.
- Preserve data fetching and routing.
- Respect motion primitives from 62-04 — do not disable motion on mobile unless reduced-motion is set.
</context>
