---
You are auditing the NFL Analytics Dashboard before a design polish phase.

**Input:**
- Live site: https://frontend-jet-seven-33.vercel.app
- 11 pages listed in the PLAN context
- Source: `web/frontend/src/app/dashboard/*/page.tsx` and corresponding feature components in `web/frontend/src/features/nfl/components/`

**Required rubric (score 0-10 per dimension per page):**
1. Typography hierarchy & readability
2. Color usage & semantic palette consistency
3. Spacing & alignment rhythm
4. Component consistency (matches shadcn/ui patterns, no one-off styles)
5. Motion & micro-interactions (is state change communicated?)
6. Empty / loading / error states
7. Mobile readiness at 375px (observed, not theoretical)
8. Information density & visual hierarchy

**Per page output (11 pages total):**
```
### Page N: /dashboard/[slug]
Overall score: X.X / 10 (weighted mean)
Sub-scores: typography X, color X, spacing X, components X, motion X, states X, mobile X, density X

Top 3 gaps (prioritized by impact × reach):
1. [specific, file-referenced gap]
2. ...
3. ...

Consistency drift vs rest of site: [notes]
```

**Cross-cutting sections (after the 11 page sections):**
- Typography drift inventory (which pages use one-off text sizes/weights)
- Color drift inventory (hardcoded hex/rgb/oklch outside theme tokens)
- Spacing drift inventory (off-grid margins/padding, inconsistent card/section gaps)
- Motion inventory (what exists today; what user actions lack feedback)
- Mobile-at-375 inventory (overflow, touch target failures, broken grids)

**Methodology constraints:**
- Read each page source AND inspect the live URL. Do not score from source alone.
- Use the `audit` skill for technical quality (a11y, responsive, theming) and the `critique` skill for UX.
- Do not propose fixes in detail here — this is a baseline, not a solution doc. A one-line "recommended direction" per gap is allowed.
- No hand-waving scores. If a dimension is unobservable on a page (e.g. no empty state visible), mark `n/a` not a fabricated number.
