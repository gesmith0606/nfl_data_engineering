### Interface contract for motion-primitives.ts

```ts
import { motion, type Variants } from 'motion/react';
import { MOTION, EASE } from '@/lib/design-tokens';

// Fade + rise for page/section entrance
export function FadeIn({ children, delay = 0, ... }: Props): JSX.Element;

// Stagger children with consistent cascade (for lists, card grids)
export function Stagger({ children, step = 0.04, ... }: Props): JSX.Element;

// Hover lift (cards, buttons)
export function HoverLift({ children, ... }: Props): JSX.Element;

// Press scale (buttons, badges)
export function PressScale({ children, scale = 0.97, ... }: Props): JSX.Element;

// Skeleton → content crossfade for data loads
export function DataLoadReveal({ loading, skeleton, children }: Props): JSX.Element;
```

All primitives MUST:
- Respect `prefers-reduced-motion: reduce` — pass-through rendering with no motion when reduced.
- Use `MOTION.base` / `MOTION.fast` and `EASE.outStandard` by default — not hardcoded numbers.
- Accept an `as` polymorphic prop where it makes sense, or forward props to an inner motion.div.
- Export TypeScript types for their props.

### Guardrails

- Motion library: `motion` ^11.18.2 (already in package.json). Do not add framer-motion, gsap, react-spring, or any other motion dep.
- Do not introduce dep on any new package.
- Do not modify files owned by 62-03 (shared layout primitives, pages 1-5, their feature components). Motion on those pages is plan 62-06's concern.
- Preserve all data-fetching, routing, and business logic.
