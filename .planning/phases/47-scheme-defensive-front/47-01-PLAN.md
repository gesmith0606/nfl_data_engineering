# Phase 47-01: Scheme Classification + Defensive Front - Plan

**Created:** 2026-04-02
**Status:** Ready

## Tasks

### T-01: Scheme Classification
- Create `src/graph_scheme.py`
- From PBP: classify each team-season run scheme based on run_gap and run_location distributions
  - Zone teams: high run_gap='end' + diverse run_location
  - Gap/power teams: high run_gap='guard'/'tackle' concentration
  - Spread/shotgun: high shotgun rate on run plays
- Create Scheme nodes with properties: scheme_type, shotgun_rate, run_left/middle/right rates, run_gap distributions
- Create RUNS_SCHEME edges (Team→Scheme) per season

### T-02: Defensive Front Profiling
- From PFR defensive weekly (type='def'): aggregate DL/LB quality per team per week
- Create DEFENDS_RUN edges with: tackles, pressures, sacks, snap_count per defender per game
- Compute team-level defensive front composite: weighted sum of DL+LB stats
- Rolling 3-game window with shift(1) temporal lag

### T-03: Scheme Matchup Features
- `def_front_quality_vs_run`: opposing DL/LB aggregate run-stop proxy (rolling 3 games)
- `scheme_matchup_score`: historical performance of this run scheme type vs this defense style
- `rb_ypc_by_gap_vs_defense`: RB efficiency by gap type against this specific defense
- `def_run_epa_allowed`: opposing defense EPA allowed on run plays (rolling 3 games)
- Pure-pandas fallback for all features
