Make `compareExternalRankings` (the 11th advisor tool and the direct owner of ADVR-03) actually work against live external sources, with a cache-backed fallback so the tool NEVER returns empty when the source is temporarily blocked. The tool already exists (commits 44f4aa9 + de93672 + 61867b5) — this plan hardens it to production quality.

The 63-01 audit is likely to flag the FantasyPros and ESPN paths as EXTERNAL_SOURCE_DOWN because those APIs are rate-limited/blocked from Railway's egress IP. Cache-first with a documented staleness flag is the production-grade answer.

Output: Hardened `external_rankings_service.py` with a three-way fallback (live → cache → empty with stale flag), plus a refresh CLI and cached JSON fixtures so the tool has a meaningful response on first deploy.
</objective>

  <files>web/api/services/external_rankings_service.py, web/api/routers/rankings.py, tests/web/test_external_rankings.py</files>
  <behavior>
    - When the live source returns 200 with parseable data: return fresh rankings, stale=false, cache_age_hours=null, update cache.
    - When the live source times out / returns 4xx/5xx: fall back to data/external/{source}_rankings.json if it exists; return stale=true, cache_age_hours=hours since file mtime. Never raise 502 to the caller.
    - When no cache exists and live fails: return empty `players: []` with stale=true, cache_age_hours=null, and a non-raising log.warning — frontend/AI is informed rather than crashed.
    - `consensus` source merges Sleeper + FantasyPros + ESPN into an average rank, skipping sources that returned nothing.
    - `rank_diff = external_rank - our_rank` (sign convention documented in the endpoint docstring).
    - Our ranks are computed from the latest Gold projections parquet using `projected_points` descending, scoped to `position` when specified.
  </behavior>
