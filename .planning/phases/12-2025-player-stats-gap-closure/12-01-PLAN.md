<objective>
Implement stats_player adapter methods with column mapping and seasonal aggregation for 2025+ player data.

Purpose: The nflverse `player_stats` release tag lacks 2025 data. A new `stats_player` tag has it under a different schema (115 columns, renamed fields). This plan adds config constants, new adapter methods, conditional routing in existing fetch methods, and tests -- so that `--data-type player_weekly --season 2025` and `--data-type player_seasonal --season 2025` work transparently through the existing CLI.

Output: Modified config.py and nfl_data_adapter.py with full test coverage in test_stats_player.py.
</objective>

  <action>
Add three private methods and modify two public methods in `NFLDataAdapter`:

**`_fetch_stats_player(self, season: int) -> pd.DataFrame`:**
- Build URL: `https://github.com/nflverse/nflverse-data/releases/download/stats_player/stats_player_week_{season}.parquet`
- Read GITHUB_TOKEN or GITHUB_PERSONAL_ACCESS_TOKEN from env; set Authorization header if present, warn if not
- Use `urllib.request.urlopen` (stdlib, no new dependency) with 60s timeout
- Read response bytes into `pd.read_parquet(io.BytesIO(data))`
- Apply column mapping via `df.rename(columns=STATS_PLAYER_COLUMN_MAP)`
- Log schema diff: count of mapped columns, count of new columns vs 2024 schema
- Return DataFrame on success, empty DataFrame on any exception (matching _safe_call pattern)

**`_aggregate_seasonal_from_weekly(self, weekly_df: pd.DataFrame) -> pd.DataFrame`:**
- Filter to `season_type == 'REG'`
- Group by `player_id`, `player_name`, `player_display_name`, `position`, `position_group`, `headshot_url`, `recent_team` (use last team), `season`
- Sum counting stat columns (attempts, completions, passing_yards, passing_tds, interceptions, sacks, sack_yards, carries, rushing_yards, rushing_tds, receptions, targets, receiving_yards, receiving_tds, etc. -- full list from RESEARCH.md SUM_COLS)
- Count distinct weeks as `games` column
- Weighted average for `dakota` (weight by attempts)
- Recalculate share columns by computing team totals then dividing: `tgt_sh = player_targets / team_targets`, etc. for `ay_sh`, `yac_sh`, `ry_sh`, `dom`, `w8dom`, `wopr_x`, `wopr_y`, `ppr_sh`, `rfd_sh`, `rtd_sh`, `rtdfd_sh`, `yptmpa`
- Read 2024 seasonal Bronze file to get exact column order; reorder output to match, filling missing columns with NaN
- Add `season_type = 'REG'` column

**Modify `fetch_weekly_data`:**
- After `_filter_seasons`, split seasons into `old_seasons` (< STATS_PLAYER_MIN_SEASON) and `new_seasons` (>= STATS_PLAYER_MIN_SEASON)
- Old seasons use existing nfl.import_weekly_data path
- New seasons loop and call `_fetch_stats_player(s)` for each
- Concat all frames and return

**Modify `fetch_seasonal_data`:**
- Same season split pattern
- Old seasons use existing nfl.import_seasonal_data path
- New seasons: call `_fetch_stats_player(s)` then `_aggregate_seasonal_from_weekly(weekly_df)`
- Concat all frames and return

Add tests with mocked HTTP responses (use `unittest.mock.patch('urllib.request.urlopen')` returning a mock parquet bytes response). Test seasonal aggregation with a fixture DataFrame covering multiple players, teams, and weeks.
