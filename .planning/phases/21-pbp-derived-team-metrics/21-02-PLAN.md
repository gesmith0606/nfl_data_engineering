  <action>
Add these functions to the "PBP-Derived Metric Functions (Phase 21)" section of `src/team_analytics.py`:

1. **`_parse_top_seconds(top_str)`** -- helper. If `pd.isna(top_str)` return `np.nan`. Split `str(top_str)` on `:`, return `float(parts[0]) * 60 + float(parts[1])`. Wrap in try/except returning `np.nan` on any parse error.

2. **`_fg_bucket(distance)`** -- helper. Returns `"short"` if distance < 30, `"mid"` if < 40, `"long"` if < 50, `"50plus"` otherwise. Returns `None` if `pd.isna(distance)`.

3. **`compute_fg_accuracy(pbp_df)`** -- receives raw PBP. Call `_filter_st_plays(pbp_df)`, then filter to `field_goal_attempt == 1`. Group by `(posteam, season, week)`. Compute:
   - `fg_att` = count of FG attempts
   - `fg_pct` = mean of `(field_goal_result == 'made').astype(int)`
   - For each bucket: apply `_fg_bucket` to `kick_distance`, then compute accuracy per bucket. Columns: `fg_pct_short`, `fg_pct_mid`, `fg_pct_long`, `fg_pct_50plus` (NaN when 0 attempts in that bucket).
   - Rename `posteam` to `team`. Note: FG is attributed to the kicking team (posteam), which is the natural ST framing.
   - Return DataFrame with (team, season, week, fg_att, fg_pct, fg_pct_short, fg_pct_mid, fg_pct_long, fg_pct_50plus).

4. **`compute_return_metrics(pbp_df)`** -- receives raw PBP. Call `_filter_st_plays(pbp_df)`.
   - **Kickoff returns**: filter `kickoff_attempt == 1`. Detect touchbacks: `return_yards == 0 AND kickoff_returner_player_id IS NULL`. For returning team (the team receiving the kick = defteam when posteam is kicking), compute `ko_return_avg` (mean return_yards excluding touchbacks) and `ko_touchback_rate` (touchbacks / total kickoffs). Group by returning team `(defteam, season, week)`, rename defteam to team.
   - **Punt returns**: filter `punt_attempt == 1`. Detect touchbacks: `punt_in_endzone == 1`. For returning team (defteam), compute `punt_return_avg` (mean return_yards excluding touchbacks) and `punt_touchback_rate`. Group by `(defteam, season, week)`, rename to team.
   - Merge kickoff and punt return metrics on `(team, season, week)`.

5. **`compute_drive_efficiency(valid_plays)`** -- receives filtered valid plays. Group by `(posteam, season, week, drive)` to get drive-level stats:
   - `plays` = count of play_id
   - `first_downs` = sum of first_down
   - `touchdowns` = sum of touchdown
   - `drive_yards` = sum of yards_gained
   Then flag `is_three_and_out = (plays <= 3) & (first_downs == 0) & (touchdowns == 0)`.
   Aggregate to team-week: `off_three_and_out_rate` = mean(is_three_and_out), `off_avg_drive_plays` = mean(plays), `off_avg_drive_yards` = mean(drive_yards), `off_drives_per_game` = nunique(drive).
   Repeat for defense grouping by `(defteam, season, week, drive)`: `def_three_and_out_rate`, `def_avg_drive_plays`, `def_avg_drive_yards`.
   Merge offense and defense on `(team, season, week)`.

6. **`compute_top(pbp_df)`** -- receives raw PBP. Filter `season_type=='REG'`, `week<=18`. Apply `_parse_top_seconds` to `drive_time_of_possession` column. Group by `(posteam, season, week, drive)` and take the max TOP seconds per drive (each play in a drive has the same drive TOP value). Then sum per `(posteam, season, week)` for `off_top_seconds`. Repeat for defteam as `def_top_seconds`. Merge on `(team, season, week)`.

All functions must have Google-style docstrings, type hints, empty DataFrame guards with correct column names, and logger.info reporting.
