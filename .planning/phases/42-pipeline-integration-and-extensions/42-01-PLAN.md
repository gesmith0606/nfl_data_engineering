    1. `_load_ship_gate(model_dir: str) -> Dict[str, str]`: Read ship_gate_report.json. For each position in report["positions"], extract verdict. If "QB" not in verdicts, check for models/player/qb/passing_yards.json on disk -- if exists, set QB to "SHIP" (per D-07 and research Pattern 2). If report file missing, log warning and return empty dict.

    2. `_is_fallback_player(row: pd.Series, min_games: int = 3) -> bool`: Return True if player should use heuristic fallback. Check: (a) all rolling columns (columns containing "roll3" or "roll6") are NaN -- rookie per D-05b, or (b) row has a "games_played" or similar count column < min_games per D-05c.

    3. `generate_ml_projections(silver_df, opp_rankings, season, week, scoring_format="half_ppr", schedules_df=None, implied_totals=None, model_dir="models/player") -> pd.DataFrame`:
       - Call _load_ship_gate(model_dir). If empty, call generate_weekly_projections and return with projection_source="heuristic".
       - For SKIP positions: filter silver_df to that position, call generate_weekly_projections, tag projection_source="heuristic".
       - For SHIP positions (QB): load models via load_player_model for each stat, load feature columns from feature_selection/{group}_features.json, assemble feature matrix from player_feature_engineering.assemble_player_features or directly from silver_df (week-1 data), call predict_player_stats to get pred_{stat} columns, rename pred_{stat} to proj_{stat} for output consistency, compute fantasy points via calculate_fantasy_points_df, tag projection_source="ml".
       - For SHIP positions: detect fallback players via _is_fallback_player, route those to heuristic.
       - Apply MAPIE intervals for SHIP QB via compute_mapie_intervals if HAS_MAPIE, else use add_floor_ceiling.
       - For heuristic positions: use add_floor_ceiling as normal (per D-16).
       - Apply injury adjustments same position in pipeline.
       - Combine all positions into single DataFrame with columns: player_id, player_name, position, recent_team, proj_{stat}... , projected_points, projected_floor, projected_ceiling, projection_source, position_rank, overall_rank.
       - Run check_team_total_coherence and log warnings (per D-09, D-10).
       - Sort by projected_points descending. Return.

    4. `check_team_total_coherence(projections, implied_totals, threshold=1.10) -> List[str]`: Per D-09/D-10. For each team in implied_totals, sum projected_points of all team players. If sum > implied_total * threshold, append warning string. Return list of warnings. No adjustment to projections (D-10).

    5. `compute_mapie_intervals(model, X_train, y_train, X_predict, alpha=0.20) -> Tuple[np.ndarray, np.ndarray, np.ndarray]`: Per D-17. Use MapieRegressor(estimator=model, method="plus", cv="prefit"). Fit on X_train/y_train (calibration only, no retrain). Predict with alpha=0.20 for 80% interval. Return (point_pred, lower_bound, upper_bound). Wrap in try/except for graceful degradation (D-18).

    6. `compute_fantasy_intervals(stat_intervals, scoring_format) -> Tuple[float, float, float]`: Per research Pattern 4. Floor uses lower bounds for positive stats, upper for interceptions. Ceiling uses upper bounds for positive stats, lower for interceptions. Score via calculate_fantasy_points_df.

    7. MAPIE import: Use try/except at module level. Set HAS_MAPIE = True/False. If False, compute_mapie_intervals returns None and caller falls back to add_floor_ceiling (D-18).
