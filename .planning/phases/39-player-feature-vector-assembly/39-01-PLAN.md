    """Player-week feature vector assembly from Silver sources.

    Joins 9 Silver data sources into per-player-per-week rows with temporal
    lag enforcement, matchup features, Vegas implied totals, and leakage detection.

    player-level granularity.

    Exports:
        assemble_player_features: Build player-week features for a single season.
        get_player_feature_columns: Return valid feature column names.
        detect_leakage: Flag features with suspiciously high target correlation.
        validate_temporal_integrity: Check shift(1) compliance on rolling features.
    """
    import glob
    import logging
    import os
    from typing import List, Optional, Tuple

    import numpy as np
    import pandas as pd

    from config import (
        PLAYER_DATA_SEASONS,
        PLAYER_LABEL_COLUMNS,
        SILVER_PLAYER_LOCAL_DIRS,
        SILVER_PLAYER_TEAM_SOURCES,
    )

    logger = logging.getLogger(__name__)

    _BASE_DIR = os.path.dirname(os.path.dirname(__file__))
    SILVER_DIR = os.path.join(_BASE_DIR, "data", "silver")
    BRONZE_DIR = os.path.join(_BASE_DIR, "data", "bronze")

    3. Join historical: rename `gsis_id` -> `player_id`, merge on `[player_id]` ONLY (no season/week — static dimension table), `how="left"`, suffix `"__hist"`, drop suffixed dups.
    4. Join defense/positional (FEAT-03): read `defense/positional`, sort by `[team, position, season, week]`, apply `shift(1)` grouped by `[team, position, season]` on `avg_pts_allowed` and `rank` columns. Rename to `opp_avg_pts_allowed` and `opp_rank`. Merge on `left_on=["opponent_team", "season", "week", "position"]`, `right_on=["team", "season", "week", "position"]`, drop extra `team` column.

After joining on `[recent_team, season, week]`, the opponent's `def_epa_per_play` is NOT automatically available. Create a separate opponent-level join: read pbp_metrics, select `[team, season, week, def_epa_per_play]`, apply `shift(1)` grouped by `[team, season]`, rename `def_epa_per_play` -> `opp_def_epa_per_play`. Merge on `left_on=["opponent_team", "season", "week"]`, `right_on=["team", "season", "week"]`, drop extra `team`.
    6. Join 5 team sources (player_quality, game_context, market_data, pbp_metrics, tendencies) on `[recent_team, season, week]` -> `[team, season, week]` using the suffix dedup pattern from `feature_engineering._assemble_team_features()`.
    7. Add implied team totals: read Bronze schedules, call `_add_implied_totals(df, schedules)`.
    8. Apply eligibility filter via `_filter_eligible_players(df)`.
    9. Exclude bye weeks per D-09: `df = df[df["week"].notna()]` (bye weeks have no usage data, so they should not appear in base, but guard anyway).
    10. Log column count and row count. Return the assembled DataFrame.
