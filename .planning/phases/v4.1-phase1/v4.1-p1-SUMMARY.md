# v4.1 Phase 1: Expand Hybrid Coverage Summary

Fixed two pre-existing bugs in the XGB SHIP path (feature file format + duplicate columns) and validated that neither RB nor QB can be routed through LGB hybrid residual -- both degrade on the sealed 2025 holdout

## Results

### Experiments Attempted

| Position | Route | 2025 MAE (before) | 2025 MAE (after) | Delta | Decision |
|----------|-------|-------------------|------------------|-------|----------|
| RB | Heuristic -> LGB Hybrid | 5.39 | 5.98 | +0.59 (+11%) | REVERT |
| QB | Heuristic -> LGB Hybrid | 8.64 | 16.15 | +7.51 (+87%) | REVERT |
| WR | LGB Hybrid (unchanged) | 4.73 | 4.73 | 0 | KEEP |
| TE | LGB Hybrid (unchanged) | 4.18 | 4.18 | 0 | KEEP |

### Bug Fixes Shipped

1. **Feature file format** (`_load_feature_cols`): Feature selection JSON files store `{"group": ..., "features": [...]}` but the loader stored the whole dict instead of extracting the features list. This caused the XGB predict path to find zero matching columns and output all-NaN predictions.

2. **Duplicate column names** (`_generate_ml_for_position`): After renaming `proj_{stat}` to `{stat}` for scoring, the DataFrame contained duplicate columns (raw + renamed), breaking pandas index alignment in `calculate_fantasy_points_df`.

These fixes surface the real XGB SHIP blocker (feature_names mismatch with QBR columns) instead of masking it with silent fallback.

## Deferred Issues

1. **XGB SHIP path feature mismatch** -- QB/RB models need `qbr_*` columns not in feature pipeline. Requires adding QBR integration to `player_feature_engineering.py` (architectural change).
2. **Duplicate rows in apply_residual_correction** -- merge on player_name creates fan-out for common names (D.Jones, J.Williams). Needs player_id-based merge or dedup guard.
3. **Walk-forward CV / production gap** -- fundamental issue where WFCV improvements don't transfer. Needs per-position regularization tuning proven on holdout before routing new positions through hybrid.

## Key Insight

Walk-forward CV improvements (-25% RB, -72% QB) did NOT predict production performance (+11% RB worse, +87% QB worse). The gap comes from: (1) production models train on all available data including test period patterns, (2) truly unseen future years have different player pools and conditions, (3) features like travel_miles and temperature drive unstable extrapolation.
