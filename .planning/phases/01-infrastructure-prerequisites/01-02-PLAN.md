Refactor the entire script. Key changes:

1. **DATA_TYPE_REGISTRY dict** at top of file. Each entry maps a data_type string to a config dict:
   ```python
   DATA_TYPE_REGISTRY = {
       "schedules": {
           "adapter_method": "fetch_schedules",
           "bronze_path": "schedules/season={season}",
           "requires_week": False,
           "requires_season": True,
       },
       "player_weekly": {
           "adapter_method": "fetch_weekly_data",
           "bronze_path": "players/weekly/season={season}/week={week}",
           "requires_week": True,
           "requires_season": True,
       },
       # ... all 15 types
   }
   ```
   Include ALL types: schedules, pbp, player_weekly, player_seasonal, snap_counts, injuries, rosters, teams, ngs, pfr_weekly, pfr_seasonal, qbr, depth_charts, draft_picks, combine.
   For types with sub-types (ngs, pfr_weekly, pfr_seasonal), add a "sub_types" key listing valid values and a --sub-type CLI arg.

2. **Local-first storage**:
   - Primary: save to `data/bronze/{bronze_path}/filename_{timestamp}.parquet`
   - Optional S3: only attempt upload if `--s3` flag is passed AND AWS credentials are valid
   - Create directories with `os.makedirs(local_path, exist_ok=True)`
   - Print local path on success

3. **Dispatch logic** replaces the if/elif chain:
   ```python
   entry = DATA_TYPE_REGISTRY[args.data_type]
   adapter = NFLDataAdapter()
   method = getattr(adapter, entry["adapter_method"])
   df = method(seasons=..., **extra_kwargs)
   ```

4. **CLI args**: Keep existing --season, --week, --data-type. Add:
   - `--s3` flag (default False) for optional S3 upload
   - `--sub-type` for NGS/PFR stat types
   - `--data-type` choices from DATA_TYPE_REGISTRY.keys()

5. **Season validation**: Use validate_season_for_type() from config before fetching.

6. Keep upload_to_s3() function for when --s3 is used, but it is no longer the primary path.
